# Launch Coordination Checklist 요약

## 핵심 원칙
- 출시 조정 체크리스트는 Google의 2005년경 원본 체크리스트로, 새 제품/서비스 출시 시 반복 가능한 신뢰성을 보장하기 위한 도구이다
- 체크리스트는 아키텍처부터 일정까지 출시의 모든 기술적 측면을 체계적으로 점검한다

## 주요 프랙티스 및 권고사항
- 아키텍처: 아키텍처 스케치, 서버 유형, 클라이언트 요청 유형, 프로그래밍 방식 클라이언트 요청
- 머신 및 데이터센터: 머신/대역폭, N+2 중복, 네트워크 QoS, DNS 부하 분산, 새 도메인 이름
- 용량/성능: HTTP 트래픽 추정, 출시 스파이크, 트래픽 믹스, 6개월 전망, 부하 테스트, 종단 간 테스트, 타 서비스 영향, 스토리지 용량
- 시스템 신뢰성/페일오버: 머신 사망/랙 실패/클러스터 오프라인/네트워크 장애 시 대응, 백엔드 사망 감지 및 대처, 클라이언트 영향 없는 종료/재시작, 부하 분산/속도 제한/타임아웃/재시도/오류 처리, 데이터 백업/복원/재해 복구
- 모니터링/서버 관리: 내부 상태 모니터링, 종단 간 동작 모니터링, 알림 관리, 모니터링의 모니터링, 재정적으로 중요한 알림과 로그
- 보안: 보안 설계 검토, 코드 감사, 스팸 위험, 인증, SSL, 출시 전 가시성/접근 제어
- 자동화/수동 작업: 서버/데이터/구성 업데이트 방법 및 변경 통제, 릴리스 프로세스, 반복 가능한 빌드, 라이브 트래픽 카나리, 단계적 롤아웃
- 성장 이슈: 여유 용량, 10배 성장 대비, 성장 알림, 확장성 병목, 선형 스케일링, 캐싱, 데이터 샤딩/리샤딩
- 외부 의존성: 서드파티 시스템, 모니터링, 네트워킹, 트래픽 볼륨, 우아한 저하, 서드파티 서비스 과부하 방지
- 일정/롤아웃 계획: 하드 데드라인, 외부 이벤트, 월요일/금요일 배포 주의, 표준 운영 절차
