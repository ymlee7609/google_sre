# Example Production Meeting Minutes 요약

## 핵심 원칙
- 프로덕션 미팅 회의록은 서비스 상태에 대한 팀의 공유된 인식을 기록하는 서비스 지향적 문서이다
- 회의록은 구조화된 의제를 따르며 장애, 페이징 이벤트, 리소스 사용, 핵심 서비스 메트릭을 체계적으로 추적한다
- 모든 액션 아이템에 담당자를 명시하여 추적 가능성을 보장한다

## 주요 프랙티스 및 권고사항
- 회의록 구조: 날짜/참석자 → 공지 → 이전 액션 아이템 리뷰 → 장애 리뷰 → 페이징 이벤트 → 비페이징 이벤트 → 모니터링 변경/침묵 → 계획된 프로덕션 변경 → 리소스 → 핵심 서비스 메트릭 → 토론/프로젝트 업데이트 → 새 액션 아이템
- Shakespeare Sonnet++ 미팅 예시:
  - 장애 리뷰: 잠재 버그(리소스 누수) + 코퍼스 미포함 소네트 + 전례 없는 트래픽으로 12.1억 쿼리 손실, 수정 배포 완료
  - 오류 예산 소진: 가용성 오류 예산 소멸로 1개월간 프로덕션 푸시 동결
  - 페이징 이벤트: AnnotationConsistencyTooEventual 알림이 주 5회 발생 - 실행 불가능 알림이므로 임계값을 60초에서 180초로 상향
  - 리소스: 인시던트 대응으로 빌린 리소스를 다음 주 반환 예정, CPU 60%, RAM 75%, 디스크 44%
  - 핵심 메트릭: 99%ile 지연 88ms(OK), 가용성 86.95%(BAD - SLO 99.99% 미달)
- 실행 불가능 알림의 처리: 근본 원인 수정이 완료될 때까지 임계값을 조정하여 노이즈를 줄임
- 계획된 변경: 클러스터 유지보수 예정 시 트래픽 자동 라우팅 확인으로 대응 불필요 확인
- 이전 액션 아이템 추적: 지속적으로 진행 상황을 검토하여 완료 여부 확인

## 핵심 인용
> "Annihilated availability error budget; pushes to prod frozen for 1 month."

## 관련 키워드
프로덕션 미팅, 회의록, 장애 리뷰, 페이징 이벤트, 오류 예산, 핵심 메트릭, 액션 아이템, 모니터링 임계값, 리소스 사용, 서비스 상태
