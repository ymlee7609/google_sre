# Example Incident State Document 요약

## 핵심 원칙
- 인시던트 상태 문서는 활성 인시던트 동안 실시간으로 유지되는 중앙 커뮤니케이션 문서이다
- 명확한 지휘 체계(Command Hierarchy)를 수립하여 인시던트 커맨더, 운영 리드, 계획 리드, 커뮤니케이션 리드를 지정한다
- 최소 4시간마다 또는 커뮤니케이션 리드 교대 시 상세 상태를 업데이트한다

## 주요 프랙티스 및 권고사항
- 문서 구조: 요약 → 상태(활성/해결) → 커맨드 포스트(IRC 채널) → 지휘 체계 → 상세 상태 → 종료 기준 → TODO 목록 → 인시던트 타임라인
- 종료 기준(Exit Criteria): 인시던트가 종료되기 위해 충족해야 할 구체적 조건 명시 - 예: 가용성 SLO 99.99% 달성, 99%ile 지연 100ms 미만을 30분 이상 유지
- 타임라인: 최신 항목이 상단에 위치하도록 역순 기록, 모든 시간은 UTC로 표기
- Shakespeare Sonnet++ 예시: 검색 인덱스에 없는 새 소네트 발견으로 인한 연쇄 장애 - 희생 클러스터로 트래픽 유도, MapReduce 인덱스 작업 수행, 글로벌 2배 용량 확보
- TODO 항목: 완료(DONE) 또는 미완료(TODO)로 명확히 상태 표시, 관련 버그 번호 연결

## 핵심 인용
> "Communications lead to keep summary updated."

## 관련 키워드
인시던트 상태 문서, 인시던트 커맨더, 지휘 체계, 종료 기준, 타임라인, 커뮤니케이션 리드, 희생 클러스터
