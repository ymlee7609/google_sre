# Understanding Adversaries 요약

## 핵심 원칙
- 적대자를 이해하려면 그들의 동기, 능력, 전술/기술/절차(TTPs)를 분석해야 한다
- 적대자는 동기에 따라 분류된다: 재미 추구자, 핵티비스트, 범죄자, 자동화 공격자, 내부자, 국가 행위자
- 위협 모델링은 누가 시스템을 공격할 가능성이 있는지, 어떤 자원과 동기를 가지고 있는지 파악하는 과정이다
- 위험 평가는 위협의 가능성과 잠재적 영향을 기반으로 우선순위를 정해야 한다
- 내부자 위협은 악의적인 경우와 실수에 의한 경우 모두 심각한 위험을 초래한다
- 공격자의 비용을 높이고 방어자의 비용을 낮추는 것이 보안 전략의 핵심이다
- 적대자는 종종 최약 링크를 공격하며, 소셜 엔지니어링은 흔한 초기 침투 방법이다
- 자동화된 공격은 규모가 크고, 이에 대한 방어도 자동화되어야 한다
- 위협 환경은 지속적으로 진화하므로 방어 전략도 적응해야 한다

## 주요 프랙티스 및 권고사항
- 프로젝트의 위험 프로파일에 맞는 위협 모델을 수립할 것 (동물 보호소 vs 증권 거래소)
- 공격 프레임워크(예: MITRE ATT&CK)를 활용하여 체계적으로 위협을 분류할 것
- 내부자 위협에 대비하여 최소 권한 원칙, 직무 분리, 다자 승인을 적용할 것
- 자동화된 공격에는 자동화된 방어(속도 제한, 봇 탐지 등)로 대응할 것
- 보안 설계 시 적대자의 관점에서 시스템을 평가하는 레드팀 활동을 수행할 것
- 공급망 공격에 대비하여 종속성의 보안을 검증할 것
- 보안 인텔리전스를 활용하여 새로운 위협 트렌드를 파악할 것
- Clifford Stoll의 사례처럼 작은 이상 징후도 추적하는 문화를 구축할 것

## 핵심 인용
> "위협 모델링의 핵심은 시스템을 공격하고자 하는 동기와 능력을 가진 적대자가 누구인지 이해하고, 그에 따라 방어 전략의 우선순위를 정하는 것이다."

> "방어자는 모든 잠재적 취약점을 방어해야 하지만, 공격자는 단 하나의 취약점만 찾으면 된다. 이 비대칭성이 보안을 어렵게 만든다."

> "내부자는 외부 공격자보다 시스템에 대한 더 많은 지식과 접근 권한을 가지고 있어 잠재적으로 더 큰 위험을 초래한다."
