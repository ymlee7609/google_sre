# Understanding Roles and Responsibilities 요약

## 핵심 원칙
- 보안은 전문가만의 영역이 아니라 **조직 전체의 책임**이다
- 개발자, SRE, 보안 엔지니어, 관리자, 임원 등 모든 역할이 보안과 신뢰성에 기여해야 한다
- 보안 전문가의 역할은 보안을 대신하는 것이 아니라, 다른 팀이 보안을 잘 수행할 수 있도록 돕는 것이다
- 조직 규모와 성숙도에 따라 보안 전문성의 깊이와 범위가 달라져야 한다
- 보안 챔피언(security champion) 모델은 중앙 보안 팀과 제품 팀 사이의 다리 역할을 한다
- Blue Team(방어/탐지)과 Red Team(공격 시뮬레이션)의 역할이 보완적이다
- 취약점 보상 프로그램(VRP)은 외부 연구자의 기여를 활용하는 효과적인 방법이다
- 보안 전문가를 채용할 때 자격증만이 아닌 실무 경험, 관심, 다양성을 종합적으로 평가해야 한다

## 주요 프랙티스 및 권고사항
- 보안을 고립된 팀에만 위임하지 말고, 조직 전체에 분산시킬 것
- 각 제품 팀에 보안 챔피언을 지정하여 중앙 보안 팀과의 협업을 촉진할 것
- 중앙 보안 팀은 디자인 리뷰, 감사, 정책/표준 수립, 안전한 프레임워크 구축에 집중할 것
- Google의 스마트 인테이크 시스템처럼 보안 자문 프로세스를 자동화할 것
- Red Team 프로그램을 운영하되, 법률/경영진의 사전 동의와 명확한 경계를 설정할 것
- Purple Team 접근(Red+Blue 협업)으로 빠르고 포괄적인 인사이트를 얻을 것
- VRP 시작 전에 기본적인 보안 리뷰와 취약점 스캐닝을 먼저 수행할 것
- 보안 비용을 영향받는 팀들에 분산시켜 비용 효율성을 높일 것

## 핵심 인용
> "보안과 신뢰성이 고립된 팀에 위임되고 그 팀이 다른 팀에 보안 관련 변경을 요구할 수 없다면, 동일한 실패가 반복될 것이다."

> "보안 전문가와 개발자가 함께 일해야 한다. 보안이 출시를 차단하는 게이트로 인식되면, 팀들은 조직적으로 우회하려 할 것이다."

> "조직의 모든 사람이 보안과 신뢰성에 책임을 지는 것이 최선의 방어이다."
