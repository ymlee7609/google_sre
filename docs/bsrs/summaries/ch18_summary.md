# Recovery and Aftermath 요약

## 핵심 원칙
- 보안 인시던트로부터의 복구는 신뢰성 인시던트와 핵심적으로 다르다: 지속적인 적대자가 존재한다
- 지속적 공격자는 복구 중에도 환경에 대한 접근을 활용하거나 언제든 재개입할 수 있다
- 복구를 수행하는 사람은 대부분 보안 전문가가 아니라 시스템을 구축하고 운영하는 엔지니어이다
- 공격자의 활동을 즉시 차단할지, 더 많은 정보를 수집하기 위해 관찰할지의 트레이드오프가 존재한다
- 복구 계획은 물류, 타임라인, 동시 실행이 필요한 작업을 포함해야 한다
- 복구 과정에서 공격자에게 경고를 주지 않으면서 시스템을 안전한 상태로 되돌려야 한다
- 사후 분석(postmortem)은 재발 방지와 조직 학습의 핵심이다
- 복구의 범위는 기술적 시스템뿐 아니라 프로세스, 정책, 조직적 변화를 포함해야 한다

## 주요 프랙티스 및 권고사항
- 복구 계획을 사전에 수립하고, 복구 단계별 체크리스트를 준비할 것
- 공격자 차단과 정보 수집 사이의 트레이드오프를 리더십과 함께 의사결정할 것
- 자격증명(비밀번호, 토큰, 키) 순환을 복구의 핵심 단계로 포함할 것
- 복구는 동시에(D-Day) 실행하여 공격자가 대응할 시간을 주지 않을 것
- 복구 후 시스템 재침입 감시를 지속할 것
- 비난 없는 사후 분석을 통해 근본 원인을 분석하고 개선 사항을 도출할 것
- 사후 분석에는 타임라인, 영향 분석, 근본 원인, 개선 조치(action items)를 포함할 것
- 개선 조치의 이행을 추적하고, 유사한 인시던트의 재발을 방지하는 시스템적 변경을 우선시할 것

## 핵심 인용
> "보안 인시던트에서 복구할 때 핵심적인 차이점은 적대자이다. 지속적인 공격자는 복구를 실행하는 동안에도 환경에 대한 접근을 활용하거나 언제든 재개입할 수 있다."

> "복구를 수행하는 사람은 대부분 보안 전문가가 아니라 영향받는 시스템을 구축하고 매일 운영하는 사람들이다."

> "공격자의 활동을 즉시 차단할 것인가, 아니면 그들의 활동을 관찰하여 더 많은 정보를 수집할 것인가? 이 트레이드오프는 비즈니스 위험과 정보 가치를 기반으로 결정해야 한다."
