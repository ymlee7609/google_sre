# Writing Code 요약

## 핵심 원칙
- 보안과 신뢰성은 소프트웨어에 사후 적용하기 어려우므로 초기 설계부터 반영해야 한다
- 강화된 프레임워크와 라이브러리를 사용하여 일반적인 보안 취약점과 신뢰성 문제의 전체 클래스를 예방한다
- 코드 단순성이 보안과 신뢰성의 기반이며, 기술 부채의 축적을 통제해야 한다
- 프레임워크는 개별 개발자가 실수하더라도 보안 속성이 유지되도록 "구조적으로 안전한(secure by construction)" 접근을 제공한다
- 공통 프레임워크의 보안 개선은 모든 의존 애플리케이션에 자동으로 전파된다
- 코드 리뷰는 보안 및 신뢰성 문제를 조기에 발견하는 핵심 메커니즘이다
- 적절한 프로그래밍 언어와 도구 선택이 보안 수준에 직접적 영향을 미친다
- 의존성 관리와 제3자 코드의 보안 검증이 중요하다

## 주요 프랙티스 및 권고사항
- RPC 프레임워크에 인증, 권한 부여, 로깅을 기본 제공하도록 설계할 것
- 타입 안전 API를 사용하여 XSS, SQL 인젝션 등 주입 취약점을 구조적으로 방지할 것
- 데드라인 전파(deadline propagation)를 구현하여 연쇄 장애를 방지할 것
- 오류 처리를 명시적으로 하고, 실패 모드를 신중하게 설계할 것
- 기술 부채를 정기적으로 정리하는 리팩토링 시간을 확보할 것
- 코드 소유권(code ownership)을 명확히 하여 변경에 대한 책임을 부여할 것
- 정적 분석 도구를 CI 파이프라인에 통합하여 자동으로 취약점을 탐지할 것
- 메모리 안전 언어(Go, Rust, Java 등)를 신뢰할 수 없는 입력 처리에 우선 사용할 것

## 핵심 인용
> "코드에는 불가피하게 버그가 포함된다. 그러나 이러한 문제 클래스에 대해 복원력이 있도록 설계된 강화된 프레임워크와 라이브러리를 사용하여 일반적인 보안 취약점과 신뢰성 문제를 피할 수 있다."

> "프레임워크는 보안 엔지니어와 SRE가 새로운 기능을 구축할 수 있는 공통 기능 표면을 제공하여 자동화 개선 기회를 만들고 전체 프로젝트 속도를 가속화한다."

> "보안과 신뢰성을 출시 후에 덧붙이는 것은 고통스럽고 덜 효과적이며, 코드베이스에 대한 다른 근본적 가정을 변경해야 할 수 있다."
