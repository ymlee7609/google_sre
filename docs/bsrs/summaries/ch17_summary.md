# Crisis Management 요약

## 핵심 원칙
- 시스템의 신뢰성은 조직이 보안 위기를 얼마나 잘 견딜 수 있는지의 척도이다
- Google은 IMAG(Incident Management at Google) 프레임워크를 사용하여 보안과 신뢰성 인시던트 모두에 동일한 방법론을 적용한다
- 위기를 인식하는 방법, 지휘권 확보, 인시던트 통제 유지가 핵심이다
- 운영 보안(OPSEC)과 포렌식은 보안 인시던트 대응의 필수 요소이다
- 커뮤니케이션은 위기 관리에서 종종 간과되지만 가장 중요한 부분 중 하나이다
- 보안 인시던트에서는 정보를 need-to-know 기반으로 제한하되, 효과적인 대응에 필요한 인원은 포함해야 한다
- 인시던트 커맨더(IC)는 명확한 지휘 체계의 정점에 서야 한다
- 위기 상황에서는 스트레스와 시간 압박 하에서 의사결정이 이루어지므로, 사전 준비된 플레이북이 필수적이다

## 주요 프랙티스 및 권고사항
- 인시던트 선언 기준을 명확히 정의하고, 팀원들이 선언을 주저하지 않도록 문화를 조성할 것
- 인시던트 커맨더, 통신 리드, 운영 리드 등 역할을 사전에 정의하고 훈련할 것
- 보안 인시던트에서는 적대자가 통신을 관찰할 수 있으므로, 안전한 통신 채널을 사용할 것
- 전쟁터(war room) 구성 시 물리적/가상 환경 모두를 고려할 것
- 교대 근무(handoff) 절차를 정립하여 24/7 대응 능력을 확보할 것
- 외부 커뮤니케이션(고객, 미디어, 규제기관)에 대한 템플릿과 절차를 사전에 준비할 것
- 인시던트 종료 후 비난 없는 사후 분석(blameless postmortem)을 반드시 수행할 것
- 정기적으로 테이블톱 연습(tabletop exercise)과 실제 훈련을 실시할 것

## 핵심 인용
> "인시던트 대응은 신뢰성과 보안 인시던트 모두에 중요하다. 우리는 보안 인시던트에 대응할 때도 동일한 방법론인 IMAG 프레임워크를 사용한다."

> "커뮤니케이션은 위기 관리의 핵심이지만 종종 간과되는 부분이다. 내부 및 외부 이해관계자에게 적절한 시점에 적절한 정보를 전달하는 것이 중요하다."

> "보안 인시던트에서는 적대자가 복구 노력을 알아차리지 못하도록 정보를 제한해야 하지만, 동시에 효과적인 대응에 필요한 모든 인원을 포함해야 하는 긴장이 존재한다."
